# Custom LazyVim Configuration Example

This directory demonstrates how to use the `configDir` option to provide custom LazyVim configuration files.

## Directory Structure

```
custom-config/
└── lua/
    ├── config/          # LazyVim config overrides
    │   ├── keymaps.lua  # Custom keymaps
    │   └── options.lua  # Custom vim options
    └── plugins/         # Custom plugin configurations
        └── colorscheme.lua

Note: Do NOT include these files in `configDir`:
  - init.lua (auto-generated by the module)
  - lazyvim.json (generated by LazyVim at runtime)
```

## Usage

In your home-manager configuration:

```nix
{
  programs.lazyvim = {
    enable = true;
    
    # Point to your config directory
    configDir = ./path/to/custom-config;
    
    # Or map a dotfiles checkout such as ~/.dotfiles/config/nvim
    # configDir = builtins.path {
    #   path = "${config.home.homeDirectory}/.dotfiles/config/nvim";
    #   name = "dotfiles-nvim-config";
    # };
    
    # Or reference from your flake
    # configDir = inputs.nix4lazyvim.packages.${system}.default + "/examples/custom-config";
    
    # Enable extras
    extras = {
      lang.nix.enable = true;
      lang.python.enable = true;
    };
    
    # Add required tools
    extraPackages = with pkgs; [
      nil
      pyright
    ];
  };
}
```

## What Gets Loaded

The module automatically handles:
1. **init.lua** - Generated by module (sets up lazy.nvim with Nix plugins)
2. **lazyvim.json** - Generated by LazyVim at runtime, so keep it out of `configDir`

Any `lazy-lock.json` you keep in `configDir` is symlinked like the rest of your configuration.

Your files from configDir:
- **lua/** - All files are symlinked to `~/.config/<appName>/lua/` (default: `~/.config/nvim/lua/`)
- **Any other directories/files** - Also symlinked (e.g., `ftplugin/`, `after/`)

## File Loading Order

LazyVim loads files in this order:
1. `init.lua` (module-generated with Nix plugins setup)
2. `lua/config/options.lua` (your custom options)
3. `lua/config/lazy.lua` (LazyVim's lazy.nvim config)
4. LazyVim core plugins + enabled extras
5. `lua/plugins/*.lua` (your custom plugins)
6. `lua/config/autocmds.lua` (your custom autocmds)
7. `lua/config/keymaps.lua` (your custom keymaps)

## Tips

- Files in `lua/config/` are for basic Neovim configuration
- Files in `lua/plugins/` should return lazy.nvim plugin specs
- You can override LazyVim defaults by returning specs with the same plugin name
- See [LazyVim documentation](https://www.lazyvim.org/configuration) for more details
